昨晚在开发寻路算法可视化的时候，我本以为生成随机迷宫是整件事里最简单的环节。写个递归回溯，随机挖墙，起点终点连起来不就完事了？结果发现要让迷宫**既保证有解**、**又有足够复杂度**、**还得好看**，难度完全不亚于实现 A* 算法本身。我花了整晚折腾这个"简单"的生成器，最后搞出了一套分层混合的策略。以下是完整的数学原理，记录一下踩过的坑。

### 1. 迷宫的图论定义

先把迷宫从直觉抽象成严格的数学对象。设网格大小为 $m \times n$，定义**节点集** $V = \{(i,j) \in \mathbb{Z}^2 \mid 0 \leq i < m, 0 \leq j < n\}$。障碍函数 $W: V \to \{0,1\}$ 标记每个格子是通道（0）还是墙（1）。

四连通邻域定义边集：

$$
E_4 = \{\{(i,j),(i',j')\} \mid |i-i'| + |j-j'| = 1\}
$$

由此得到网格图 $G_{grid} = (V, E_4)$。我们的目标是构造子图 $G_{maze} = (V_{open}, E_{pass})$，其中 $V_{open} = \{v \in V \mid W(v) = 0\}$，$E_{pass} \subseteq E_4$ 是实际可通行的边。

对于演示用的迷宫，必须满足三个硬约束：

1. **可解性**：给定起点 $s$ 和终点 $t$，存在路径 $P(s,t) \subseteq G_{maze}$
2. **边界封闭**：$\forall v \in \partial V \setminus \{s,t\}, W(v) = 1$，其中 $\partial V$ 是四周边界
3. **局部复杂性**：避免大面积开放区域（块状空白）和过度狭窄的死胡同

### 2. 阶段1：递归回溯生成完美迷宫

递归回溯算法（Recursive Backtracker）本质上是在 $G_{grid}$ 上构造**生成树**（Spanning Tree）。算法从起点 $s$ 出发，维护栈 $S$ 和已访问集合 $Visited$。

**算法 2.1（递归回溯）**

```
1. 初始化：∀v∈V, W(v)←1（全部为墙）
2. 选择起始节点 s，标记 W(s)←0，Visited←{s}，栈 S←[s]
3. while S ≠ ∅ do
4.   current ← S.top()
5.   Neighbors ← {u ∈ N(current) | u ∉ Visited ∧ 在边界内}
6.   if Neighbors ≠ ∅ then
7.     next ← RandomSelect(Neighbors)
8.     打通 current 与 next 之间的墙（标记 W(next)←0）
9.     Visited ← Visited ∪ {next}
10.    S.push(next)
11.  else
12.    S.pop()
```

**定理 2.1（生成树性质）**  算法 2.1 生成的子图 $T = (V_{open}, E_T)$ 是 $G_{grid}$ 的生成树，且 $|E_T| = |V_{open}| - 1$。

*证明*：首先证连通性。算法通过 DFS 遍历，每个新节点通过单条边与已访问集合连接。由于网格连通且 DFS 会访问所有从 $s$ 可达节点，$T$ 连通。

再证无环性。假设存在环 $C = (v_1, v_2, \dots, v_k, v_1)$。设 $v_i$ 是环中最后被访问的节点（按压栈时间戳）。$v_i$ 加入时，其环上邻居 $v_{i-1}$ 和 $v_{i+1}$ 至多一个被访问（否则 $v_i$ 不会是最晚的）。不妨设 $v_{i-1} \in Visited$ 而 $v_{i+1} \notin Visited$。但边 $\{v_i, v_{i+1}\}$ 只有在 $v_{i+1}$ 被访问时才加入 $E_T$，矛盾。故 $T$ 无环。

由树的性质，$|E_T| = |V_{open}| - 1$。这意味着通道数恰好比开放节点数少一，任意两点间路径唯一。

这种"完美迷宫"虽然保证了可解性，但**路径复杂度为零**——起点到终点只有一条路。跑可视化的时候，A* 和 BFS 的搜索轨迹完全重合，根本看不出启发式函数的裁剪效果。这显然不行。

### 3. 阶段2：引入环路的拓扑扰动

我需要打破唯一路径的特性。在图论上，这意味着要把树变成**含环图**（Cyclic Graph）。具体做法是在阶段1的树 $T$ 基础上，以密度 $\rho$ 随机添加边 $E_{loop}$，形成 $G' = (V, E_T \cup E_{loop})$。

候选边集来自树补图：$E_{candidate} = E_4 \setminus E_T$。但随机添加边有个大坑：如果不对节点的开放邻居数做限制，很容易形成 $2\times2$ 的块状空白区域，迷宫感瞬间消失。

**局部密度约束**：对于候选边 $e = \{u,v\}$，仅当两端点的开放邻居数满足 $1 \leq \deg_W(u) \leq 3$ 且 $1 \leq \deg_W(v) \leq 3$ 时才打通。这里 $\deg_W(v) = |\{u \in N(v) \mid W(u) = 0\}|$。

**定理 3.1（连通性保持）**  在约束 $1 \leq \deg_W(v) \leq 3$ 下添加边，图 $G'$ 保持连通，且不会生成度数为1的新叶子节点（除初始边界外）。

*证明*：原树 $T \subseteq G'$，故 $G'$ 连通。度数约束防止了星型结构的产生：若允许 $\deg_W(v) > 3$，某些节点会成为高度 hub；若允许 $\deg_W(v) < 1$，则生成新的死胡同。约束保证每个开放区域都有合理的分支数。

添加边后，图的**圈秩**（Cyclomatic Number，独立环路数）为：

$$
\mu(G') = |E_T \cup E_{loop}| - |V| + 1 = |E_{loop}|
$$

这意味着每添加一条边就恰好增加一个独立环路。设 $|V| = N$，则 $\mu(G') \approx \rho N$。这个量直接决定了寻路算法的**决策点**数量：每经过一个环路，算法就要在多条路径中选择。

**参数选择**：我最初设 $\rho = 0.08$，结果环路太多，加上后续的洞穴簇，经常连成大片开放区域。修复后降到 $\rho = 0.04$，并将约束收紧为 $1 \leq \deg_W(v) \leq 3$。这禁止了**完全子图** $K_4$ 的形成（四个节点互相连接，度数均为2或3），确保环路呈现"瓶颈"结构——即环路通过狭窄通道连接，而非大面积开放区域。

### 4. 阶段3：边界约束与双层墙修复

在初始实现中，我忽略了网格尺寸的奇偶性对生成树边界的影响。当使用步长为 2 的跳跃（标准做法，用于保证墙与通道的交替），当 $m$ 或 $n$ 为偶数时，算法会在 $(m-2, \cdot)$ 或 $(\cdot, n-2)$ 处停止，留下一层未被触及的单元格，与强制设置的最外层边界形成**双层墙**：

$$
\exists y: W(m-2, y) = 1 \land W(m-1, y) = 1
$$

这在视觉上表现为迷宫外围有两圈黑边，压缩了有效搜索空间。

**修复策略的数学表述**：

1. **生成阶段的严格内含**：将递归回溯的遍历域收紧为 $[1, m-2] \times [1, n-2]$，即要求：

   $$
   \forall (x,y) \in \text{Traversal}: 1 \leq x < m-1 \land 1 \leq y < n-1
   $$

   这确保了生成树完全位于内部区域 $\Omega = [1, m-2] \times [1, n-2]$，最外层边界由独立的初始化代码统一设置为墙。
2. **后处理打通的概率模型**（`fixDoubleBoundary`）：  
   对于紧邻边界的内部墙单元 $v \in \{(1,y), (m-2,y), (x,1), (x,n-2)\}$，若其内侧邻居 $u$ 满足 $W(u)=0$，则以概率 $p$ 打通：

   $$
   P(W(v) \to 0) = \begin{cases} 
   0.5 & \text{if } v \in \text{Right/Bottom} \\
   0.3 & \text{if } v \in \text{Left/Top}
   \end{cases}
   $$

   左右/上下概率差异化基于起点 $s$ 位于左上、终点 $t$ 位于右下的布局：左上区域已有较高的开放度（$s$ 的出口），较低的概率防止过度开放。

**定理 4.1（边界紧致性）**  在修复策略下，搜索空间 $\Omega$ 与墙 $\partial\Omega$ 的接触面为单一层，且 $\partial\Omega$ 形成闭合曲线。

*证明*：由生成阶段的严格内含，生成树不触及 $x=m-1$ 或 $y=n-1$。后处理仅打通与内部通道相邻的边界墙，且保持 $W(m-1,y)=1$（最外层）。因此边界为单层，且四边闭合，仅 $s,t$ 处开口。

### 5. 连通性保证：双向BFS的数学完备性

尽管阶段1的生成树保证了连通性，但阶段2的随机扰动和边界修复可能意外切断起点 $s$ 与终点 $t$ 的连接。因此需要**后验的连通性验证与修复**。

![image](/blog/assets/image-20260223125226-y6n8sdo.png)

**算法 5.1（双向BFS连通性修复）**

```
1. 初始化：F_s ← {s}, Visited_s ← {s}；F_t ← {t}, Visited_t ← {t}
2. while F_s ≠ ∅ 或 F_t ≠ ∅ do
3.   交替扩展 F_s 和 F_t：
4.   F_s ← F_s ∪ N(F_s) \ Visited_s（只访问 W=0 的节点）
5.   F_t ← F_t ∪ N(F_t) \ Visited_t
6.   if F_s ∩ F_t ≠ ∅ then
7.     找到汇合点 v，返回路径 Path_s(s,v) ∪ Path_t(v,t)
8. if 队列为空且未汇合 then
9.   选取 u ∈ F_s, v ∈ F_t 使 Manhattan(u,v) 最小
10.  打通 Path_L(u,v) 上所有墙（L形最短路径）
```

**定理 5.1（双向BFS的最优性）**  设 $d(s,t)$ 为最短路径长度，双向BFS找到的第一个汇合点 $v \in F_s \cap F_t$ 满足 $d(s,v) + d(v,t) = d(s,t)$。

*证明*：BFS 按层扩展。当 $v$ 首次被两边同时访问时，$d(s,v) = k_1$，$d(t,v) = k_2$，且 $k_1 + k_2 = d(s,t)$（因为两边扩展的层数之和等于最短路径长度）。若存在更短路径，则 $v$ 应更早被访问，矛盾。

**定理 5.2（修复的完备性）**  若双向BFS未找到路径，执行第9-10步后，$s$ 与 $t$ 必然连通。

*证明*：若BFS失败，说明 $s$ 和 $t$ 位于不同连通分量 $C_s$ 和 $C_t$。选取 $u \in C_s, v \in C_t$ 使 Manhattan 距离最小，打通它们之间的L形路径 $\gamma$。$\gamma$ 连接 $u$ 和 $v$，故 $C_s$ 和 $C_t$ 通过 $\gamma$ 合并为一个连通分量，$s$ 与 $t$ 连通。

**复杂度分析**：双向BFS将搜索空间从 $O(b^d)$ 降至 $O(b^{d/2})$，其中 $b$ 为分支因子，$d$ 为路径长度。在迷宫这种稀疏图中（$b \approx 2$），实际运行时间接近线性 $O(N)$。

### 6. 洞穴簇的移除与启发式失真

在最初的分层策略中，我尝试以 $\rho_c=0.02$ 的密度添加 $2\times2$ 或 $3\times3$ 的洞穴簇。这相当于在网格上叠加**矩形随机几何图**。然而，当这些簇与阶段2的环路开口重叠时，会合并成大面积的开放区域，破坏迷宫的"狭长通道"特性。

![image](/blog/assets/image-20260223125134-jikj59n.png)

**修复策略**：完全移除独立的洞穴簇生成步骤，将非结构化扰动收敛到环路引入阶段。通过强化局部密度约束（$1 \leq \deg_W(v) \leq 3$）和降低开口密度（$\rho=0.04$），避免了块状空白的形成。

这种简化实际上更符合**半结构化环境**的定义：底层是确定性的树状骨架（保证可解性），中层是稀疏的随机环路（提供多路径决策），没有额外的混沌层。对于 A* 算法的启发式函数 $h(n) = |x_n - x_t| + |y_n - y_t|$，这种结构仍然能制造**启发式误差**：

$$
\epsilon(n) = h(n) - h^*(n)
$$

在环路区域，由于存在隐藏的绕路，实际代价 $h^*(n)$ 可能大于曼哈顿距离，导致 $\epsilon(n) > 0$（过估计），这会破坏 A* 的一致性（Consistency），使得算法需要扩展更多节点来修正估计，从而在可视化中展示出明显的"犹豫"行为。

### 7. 最终参数配置与总结

经过反复调试，最终的参数配置如下：

|参数|值|作用|
| ---------------| -------------------| -------------------------------------|
|环路密度 $\rho$|$0.04$|圈秩 $\mu(G) \approx 0.04N$，平衡路径多样性与开放区域控制|
|邻居约束 $\deg_W$|$[1,3]$|禁止 $K_4$ 子图，防止块状空白|
|边界打通概率 $p$|$0.3$（左上）/$0.5$（右下）|基于起点/终点位置的空间差异化|
|起点/终点位置|$(1,1)$ 和 $(m-2,n-2)$|避开边界，确保在生成域内部|

生成的图 $G_{final}$ 满足：

- **稀疏性**：$|E| \approx 1.04|V|$（树边 $|V|-1$ 加 4% 额外边）
- **局部复杂性**：最大度数 $\Delta(G) \leq 4$，最小度数 $\delta(G) \geq 1$（除边界外）
- **可解性**：通过双向BFS后验保证，时间复杂度 $O(N)$

这套分层混合策略最终给出的，是一个**半结构化环境**：底层是确定性的树状骨架保证可解性，中层是严格控制的随机环路提供路径选择，边界是单层的紧致封闭。它既不是纯粹的规则网格，也不是完全随机的噪声地形，而是介于两者之间的、具备数学可控复杂度的测试平台。

现在我的可视化工具终于可以稳定生成"看起来顺眼、算起来有挑战、绝对不会卡死"的迷宫了。
