# SAKURAIN Tech - AI Coding Agent Guide

## Project Overview

SAKURAIN Tech is a personal portfolio and blog website built with React, TypeScript, and Vite. It features a modern, animated UI with 3D visualizations, a blog system, documentation, algorithm visualizations, and developer tools.

**Website URL**: https://sakurain.net

### Key Features

- **Homepage**: Personal portal with ASCII art logo, tech stack evolution charts, and service showcases
- **Blog**: Markdown-based blog with tags, archives, RSS feeds, and comment system
- **Docs**: Documentation system with course materials and technical articles
- **Algorithm Visualizer**: Interactive algorithm visualization (sorting, graph algorithms, dynamic programming)
- **Developer Tools**: Collection of utility tools (JSON formatter, Base64, color converter, etc.)
- **3D Earth Visualization**: Cesium-based globe with satellite orbit visualization
- **Notes**: Micro-blogging/note-taking system with mood tracking

## Code Style Guidelines

### TypeScript Conventions

- **Strict mode enabled**: All strict TypeScript options are on
- **Path aliases**: Use `@/` prefix for imports from `src/`
  ```typescript
  import { Button } from '@/components/ui/button';
  import { useTheme } from '@/hooks';
  ```
- **Type imports**: Use `type` keyword for type-only imports
  ```typescript
  import type { SiteData } from '@/types';
  ```

### Component Structure

- Use functional components with hooks
- Props interfaces should be defined inline or in `types/` directory
- shadcn/ui components use `cn()` utility for class merging
- Animation components should respect `usePrefersReducedMotion()`

### Naming Conventions

- **Components**: PascalCase (e.g., `BlogCard.tsx`)
- **Hooks**: camelCase with `use` prefix (e.g., `useTheme.ts`)
- **Utilities**: camelCase (e.g., `utils.ts`)
- **Constants**: SCREAMING_SNAKE_CASE for true constants
- **CSS Classes**: Use Tailwind utility classes; custom classes in kebab-case

### File Organization

- Each major feature has its own directory under `src/pages/`
- Shared components go in `src/components/`
- Feature-specific components stay in their page directory
- Re-export from `index.ts` files for cleaner imports

## Edge Functions (API)

Location: `edge-functions/`

The project uses EdgeOne/Cloudflare Edge Functions for serverless APIs.

### Security Model

- **HMAC-SHA256** authentication for write operations
- **Timestamp validation** (5-minute tolerance)
- **Nonce validation** to prevent replay attacks
- **Rate limiting** (60 requests/minute per IP)
- **KV storage** for nonces and data

### API Endpoints

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/api/danmaku/list` | GET | No | Get satellite danmaku list |
| `/api/danmaku/add` | POST | Yes | Add new danmaku |
| `/api/comments` | GET | No | Get comments for a post |
| `/api/comments` | POST | Yes | Post a comment |

### Environment Variables for Edge Functions

```env
API_SECRET_KEY=your-32-byte-secret-key
KV_SECRET=auth-nonces-namespace
DANMAKU_KV=danmaku-namespace
COMMENTS_KV=comments-namespace
RATE_LIMIT_KV=rate-limit-namespace
```

## Performance Considerations

### Build Optimizations

- Code splitting with manual chunks (react-vendor, framer-motion, three-core, etc.)
- CSS code splitting enabled
- esbuild minification with console/debugger removal
- Source maps disabled in production

### Runtime Optimizations

- **Route-based lazy loading** - Pages loaded on demand
- **Staggered effect loading** - Animations start at different times
- **Performance context** - Detects device capability and adjusts quality
- **Reduced motion support** - Respects `prefers-reduced-motion`
- **GPU acceleration** - Uses `transform` and `will-change` appropriately

### Image Handling

- Images stored in `public/image/`
- WebP format preferred
- Cesium assets in `public/Assets/`
- Map data in `public/map-data/`

## Common Development Tasks

### Adding a New Page

1. Create directory under `src/pages/MyPage/`
2. Create `index.tsx` with page component
3. Add lazy import in `src/main.tsx`
4. Add route in `<Routes>` component
5. Add to navigation in `public/data/site-data.json` if needed

### Adding a New Tool

1. Create tool component in `src/pages/Tools/tools/my-tool.tsx`
2. Export from `src/pages/Tools/tools/index.ts`
3. Register in `src/pages/Tools/registry.ts`

### Adding a Blog Post

1. Create markdown file in `public/blog/posts/`
2. Add proper frontmatter
3. Run `npm run build` to regenerate indices
4. Images go in `public/blog/assets/`

### Adding a UI Component

1. Use shadcn/ui CLI or create manually in `src/components/ui/`
2. Follow existing component patterns
3. Use `cn()` utility for class merging
4. Export from `src/components/ui/` index if shared

## Troubleshooting

### Build Failures

- Check Node.js version (20.18.0+ recommended)
- Delete `node_modules` and reinstall if dependency issues
- Ensure all environment variables are set for full build

### Type Errors

- Run `npx tsc --noEmit` to check TypeScript errors
- Check `tsconfig.app.json` for strict settings

### Edge Function Issues

- Verify KV namespace bindings
- Check `API_SECRET_KEY` is configured
- Review EdgeOne console logs
